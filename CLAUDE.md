# Portfolio Project — Claude Rules

## How Claude Should Work on This Project

1. **Explain before changing** — Always describe what you are about to change before editing any files.

2. **Summarize after changing** — After editing files, list exactly which files changed and what was changed in each one.

3. **Use beginner-friendly language** — Avoid jargon. When technical terms are needed, explain them in plain English.

4. **Avoid large rewrites** — Make small, focused changes. Do not rewrite large sections of code unless explicitly asked.

5. **Keep the website visually polished** — The site should always look clean, modern, and portfolio-ready. Visual quality matters.

6. **Use plain HTML, CSS, and JavaScript** — Do not use React, Next.js, Tailwind, or any complex frameworks unless asked. Keep the stack simple.

7. **Comment the code** — When adding code, include comments that explain what the important parts do and why.

8. **Explain errors clearly** — If something breaks, explain what caused it, which file is involved, and exactly how it was fixed.

9. **Ask before deleting** — Never delete any file, section, or block of code without asking first.

10. **Preview instructions after every major step** — After completing a significant change, remind the user how to preview the website locally (e.g., open `index.html` in a browser).
